## 🔐 文章一：Shannon - AI渗透测试的新范式

### 原文信息
- **项目地址**: [https://github.com/KeygraphHQ/shannon](https://github.com/KeygraphHQ/shannon)
- **开发团队**: Keygraph HQ
- **许可证**: AGPL-3.0

### 核心概述

Shannon是一款**全自动AI渗透测试工具**，其设计理念是"在别人之前攻破你的Web应用"。它不仅仅是一个漏洞扫描器——它会**实际执行攻击**来证明漏洞的可利用性。

在XBOW基准测试（无提示、源码感知模式）中，Shannon达到了**96.15%的成功率**。

### 为什么这很重要？

当前开发流程存在一个巨大的安全缺口：

```
开发团队：每天用Claude Code/Cursor发布代码
渗透测试：一年一次

= 364天的安全盲区
```

Shannon通过提供**按需白盒渗透测试**来填补这一空白。

### 技术架构：模拟人类渗透测试者

Shannon采用四阶段架构，完美模拟专业渗透测试人员的工作流程：

```
         ┌──────────────────────┐
         │   1. 侦察阶段        │  ← 扫描攻击面、分析代码
         └──────────┬───────────┘
                    │
         ┌──────────┴───────────┐
         │     2. 漏洞分析       │  ← 并行分析多种漏洞类型
         │   (Injection/XSS等)   │
         └──────────┬───────────┘
                    │
         ┌──────────┴───────────┐
         │     3. 漏洞利用       │  ← 实际执行攻击证明漏洞
         │   (浏览器自动化)      │
         └──────────┬───────────┘
                    │
         ┌──────────┴───────────┐
         │     4. 报告生成       │  ← 只报告已验证的漏洞
         └──────────────────────┘
```

**关键原则**: "No Exploit, No Report" —— 如果无法成功利用，就不报告为漏洞（消除误报）。

### 核心能力

1. **全自动化运行**
   - 单命令启动：`./shannon start URL=https://your-app.com REPO=/path/to/repo`
   - 支持2FA/TOTP登录、Google登录
   - 零人工干预

2. **覆盖的漏洞类型**
   - SQL注入 (Injection)
   - 跨站脚本 (XSS)
   - 服务器端请求伪造 (SSRF)
   - 身份认证/授权绕过

3. **集成安全工具链**
   - Nmap（端口扫描）
   - Subfinder（子域名发现）
   - WhatWeb（技术栈识别）
   - Schemathesis（API测试）

### 实战成果

**OWASP Juice Shop测试结果**：
- 发现 **20+** 高危漏洞
- 完成身份认证绕过，导出整个用户数据库
- 创建新管理员账户（权限提升）
- 发现SSRF漏洞，实现内网探测

**c{api}tal API测试结果**：
- 发现 **15** 个关键漏洞
- 通过命令链接绕过黑名单实现root级注入
- 利用遗留v1 API端点完全绕过认证

### 技术细节

Shannon基于**Anthropic Agent SDK**构建，推荐使用Claude模型：

```bash
# 快速开始
git clone https://github.com/KeygraphHQ/shannon.git
cd shannon

export ANTHROPIC_API_KEY="your-api-key"
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=64000

./shannon start URL=https://your-app.com REPO=/path/to/repo
```

**成本与时间**：
- 运行时间：1-1.5小时
- 预估成本：约$50 USD（使用Claude 4.5 Sonnet）

### 深度思考

**Shannon代表的趋势**：

1. **AI安全工具的成熟化** - 从辅助扫描到自主渗透，AI安全能力正在质变
2. **Proof-by-Exploitation范式** - 通过实际利用证明漏洞，而非静态分析猜测
3. **开发与安全的融合** - CI/CD集成让安全测试成为开发流程的一部分

**潜在风险**：同样的技术也可能被恶意使用。这是一把双刃剑。

## 🔄 文章二：DyTopo - 让Agent学会"动态组队"

### 原文信息
- **论文标题**: DyTopo: Dynamic Topology Routing for Multi-Agent Reasoning via Semantic Matching
- **论文地址**: [https://arxiv.org/abs/2602.06039](https://arxiv.org/abs/2602.06039)
- **提交时间**: 2026年2月5日
- **目标会议**: ICML

### 核心问题

现有多Agent系统存在一个根本性问题：

> **通信拓扑是固定的** —— Agent之间谁和谁交流，在整个任务过程中保持不变。

但实际推理过程是**阶段性的**：
- **早期**：需要广泛探索、共享问题框架
- **后期**：需要精确的、有针对性的信息交换来诊断错误

用同一个通信模式处理所有阶段，效率很低。

### DyTopo的解决方案

DyTopo引入了**动态拓扑重构**机制：

```
传统方式: 固定通信图 → 所有轮次复用
DyTopo:   每轮重建通信图 → 根据当前目标动态调整
```

核心创新是**Query-Key语义匹配**：

```
每个Agent在每轮输出两个描述符：
- Query（需求）: "我现在需要什么信息"
- Key（能力）: "我能提供什么信息"

DyTopo通过语义相似度匹配，决定谁向谁发送消息
```

### 技术细节

**语义对齐量化**：

使用预训练的句子编码器（all-MiniLM-L6-v2）将Query/Key嵌入向量空间：

```
q_i = Embed(query_i)   # Agent i 的需求向量
k_j = Embed(key_j)     # Agent j 的能力向量

r_{i,j} = cosine_similarity(q_i, k_j)  # 语义相关性得分
```

**稀疏图构建**：

通过阈值τ控制图的稀疏度：

```
Edge(j→i) exists iff r_{i,j} > τ_edge
```

这意味着只有当Agent j的能力真正匹配Agent i的需求时，才建立通信链接。

### 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                     Manager Agent                        │
│    - 设定每轮目标 (Round Goal)                           │
│    - 决定是否终止                                        │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                    Worker Agents                         │
│                                                          │
│   Round t:                                               │
│   1. 每个Agent根据当前目标生成Query/Key描述符            │
│   2. 语义匹配构建本轮通信图 G(t)                         │
│   3. 私有消息沿图边路由                                  │
│   4. 更新Agent记忆，进入下一轮                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 实验结果

在4个LLM骨干网络上，跨代码生成和数学推理任务，DyTopo**在所有16个设置中都取得最佳**：

| 基准测试 | 提升幅度 |
|---------|---------|
| HumanEval (代码) | +9.14% (Qwen3-8B) |
| APPS-Competition | +9.11% (GPT-oss-120B) |
| Math-500 | +17.14% (Llama3-8B) |
| Omni-Math | +15.72% (Qwen3-8B) |

**平均提升：+6.2%**

### 关键发现

1. **通信轮次存在最优值**
   - HumanEval在第5轮达到峰值（92.07%）
   - Math-500在第9轮达到峰值（87.14%）
   - 更多轮次不一定更好！

2. **拓扑随任务阶段演变**
   - 早期：密集图，广泛探索
   - 中期：选择性连接，聚焦验证
   - 后期：稀疏图，最小依赖，组装答案

3. **稀疏度阈值是关键超参**
   - τ=0.3 对APPS最优
   - τ=0.4 对Omni-Math最优
   - 不同任务需要不同的通信密度

### 深度思考

**DyTopo的启示**：

1. **通信即计算** - Agent之间的信息流动方式，本身就是一种可优化的计算结构

2. **可解释性bonus** - 动态拓扑图为多Agent协作提供了清晰的"协调轨迹"，便于调试和分析

3. **自组织协作** - Agent不再是被动接受固定角色，而是根据当前需求动态重组协作网络

**与强化学习的联系**：这种Query-Key匹配机制类似于注意力机制，但应用在Agent层面而非Token层面。
